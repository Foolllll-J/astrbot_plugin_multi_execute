## ✨ 简介

一款为 [**AstrBot**](https://github.com/AstrBotDevs/AstrBot) 设计的指令模拟器。支持免唤醒触发指令、连续执行指令和模拟其他用户执行指令。

---

## 🌟 功能特性

- **免唤醒触发**：无需输入唤醒前缀即可触发指令。
- **连续执行**：支持将一个指令连续执行多次。
- **用户模拟**：支持模拟其他用户执行指令。
- **安全限制**：可配置单次最大执行次数，防止滥用。
- **前缀模式**：支持配置是否强制要求用户携带指令前缀。
- **权限管理**：支持白名单模式，仅允许特定用户使用。

---

## 🚀 使用方法

### 指令格式

#### 1. 免唤醒指令

在插件配置中添加指令后，用户可以直接发送指令来执行，无需输入唤醒前缀。

- **示例**：添加指令 `签到`
- **使用**：直接发送 `签到` 或 `签到 参数`
- **效果**：Bot 会自动添加唤醒前缀，转换为 `/签到` 或 `/签到 参数`

> 💡 **提示**：开启 `所有指令免唤醒` 模式后，所有已注册的指令都无需前缀即可触发。可通过 `需唤醒指令列表` 设置黑名单。

#### 2. 连续执行指令

使用 `nx 指令` 格式，其中 `n` 为执行次数，`nx`前无需指令前缀。

- **示例**：`3x /签到` (前缀模式开启时) 或 `3x 签到` (前缀模式关闭时)
- **效果**：Bot 将连续执行 3 次 `/签到` 指令。

#### 3. 模拟用户指令

使用 `/模拟 @用户 指令` 格式，允许管理员模拟其他用户执行指令。

- **示例**：`/模拟 @用户A /查询状态`
- **效果**：Bot 将以用户A的身份执行 `/查询状态` 指令。

## ⚙️ 配置说明

在 AstrBot 管理面板中，你可以找到以下配置项：

| 配置项 | 说明 | 默认值 |
| :--- | :--- | :--- |
| **使用者白名单** | 填入用户 ID 列表，不填表示所有人可用。Bot 管理员默认可用。 | `[]` |
| **前缀模式** | 开启：用户必须自己带前缀（如 3x /签到）。关闭：用户可不带前缀，插件自动补全；若用户已带前缀则不补全。 | `False` |
| **执行间隔时间 (秒)** | 连续执行指令之间的间隔时间。 | `1` |
| **最大执行次数限制** | 单次指令连续执行的最大次数限制。 | `10` |
| **显示开始执行提示** | 开启后，模拟执行时会发送一条提示消息（如：开始连续执行 3 次...）。 | `False` |
| **所有指令免唤醒** | 开启后，所有已注册的指令都无需前缀即可触发。 | `False` |
| **免唤醒指令列表** | 无需唤醒前缀即可触发的指令列表。 | `[]` |
| **需唤醒指令列表** | 在这些指令即使在所有指令免唤醒模式下，仍然需要前缀。 | `[]` |
| **免唤醒白名单群组** | 配置允许使用免唤醒触发的群组，为空表示全局可用。 | `[]` |
